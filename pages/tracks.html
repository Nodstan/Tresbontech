Two Tracks,
One Destination

Every student arrives at AI education from a different starting point. Neither track is better — they require different entry doors. Tresbontech tutors read which track a student belongs to within the first session.

The placement rule: If a student can write a function independently and has built at least one small project, they are Track A. If they cannot, they are Track B. Assess this in the first session through a 5-minute conversation — not a test. Never make a student feel "behind" for being in Track B.
Track A — Code First
Already Writes Code

Students with at least 3 months of Python or similar. They can write functions independently, have built small projects, and ask "how does this work?" before "what can this do?"

What they learn
Python for AI
NumPy, Pandas, scikit-learn — real data science tools
Model Training Concepts
How machines learn from data — theory meets code
APIs & Pre-Built Models
OpenAI, HuggingFace, Google AI — real production tools
Build & Deploy
Simple web apps using AI APIs with Streamlit
Prompt Engineering in Code
Structured prompting inside programs — system prompts, few-shot
Evaluation & Ethics in Code
Testing models for bias — technical fairness metrics
Who they areStudents who have completed at least 3 months of Python, Scratch (transitioning to text), or any text-based coding. They can write a function without help. They debug independently. They get frustrated when things are too easy.
Track B — Concept First
New to Coding

Zero to minimal coding experience. Entry through AI concepts, ethics, and no-code tools. They learn to work powerfully with AI before learning to build it — and this is a completely valid, increasingly valuable skill set.

What they learn
What Is AI?
Demystifying myths vs reality — accurate mental models first
How AI Learns
Training data, patterns, prediction — concepts without code
No-Code AI Tools
Teachable Machine, ML5.js, ChatGPT — real AI, zero code
AI in Everyday Life
Spotting AI in apps students already use every day
Prompt Engineering
Getting excellent, reproducible results from AI tools
Ethics & Critical Thinking
Bias, fairness, misinformation — responsible AI citizenship
Who they areStudents with zero to minimal coding experience. Often creative thinkers, writers, or artists. They ask "what can this do?" and "is this fair?" They respond strongly to real-world application and visible results over syntax and debugging.
The Migration Journey

Many students will start in Track B and migrate to Track A over 6–12 months as their confidence grows. This is the ideal journey — understand AI conceptually first, then build with it. Never rush this. The migration happens naturally when students become curious about "how" rather than just "what." That curiosity is the signal.
